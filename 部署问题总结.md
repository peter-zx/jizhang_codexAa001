## 二、遇到的所有问题及解决方案

### 问题 1: SSH 连接超时（代理问题）
**现象**: paramiko 连接 ECS 时 TimeoutError
**原因**: SOCKS5 代理 127.56.100.1:7890 不稳定
**解决**: 
- 方式一：使用代理 `ssh -o ProxyCommand="nc -X 5 -x 127.0.0.1:7890 %h %p"`
- 方式二：直接连接（代理不可用时）

### 问题 2: GitHub 克隆失败
**现象**: `git clone` 超时或连接重置
**原因**: 代理不稳定或 GitHub 访问受限
**解决**: 改用 SFTP 直接上传本地代码文件

### 问题 3: Python 模块导入错误
**现象**: `ModuleNotFoundError: No module named 'app'`
**原因**: 
- ECS 上代码直接放在根目录，没有 app/ 子目录
- 或 PYTHONPATH 未设置
**解决**: 
```bash
export PYTHONPATH=/root/jizhang_codexAa001
# 启动命令
python -m uvicorn app.main:app
```

### 问题 4: 依赖缺失（逐个安装）
**现象**: 启动时各种 ModuleNotFoundError
**解决**: 逐个安装缺失包
```bash
pip install fastapi uvicorn sqlalchemy python-multipart jinja2 openpyxl itsdangerous python-docx
```

### 问题 5: Jinja2 + Starlette 版本不兼容 ⭐关键问题
**现象**: 
```
TypeError: unhashable type: 'dict'
# 发生在 TemplateResponse 调用时
```
**原因**: 
- Jinja2 3.0.3 + Starlette 1.0.0 不兼容
- `TemplateResponse` 的 API 变了
**解决**: 修改 `app/main.py` 中的 render 函数
```python
# 旧代码（报错）
return templates.TemplateResponse(name, payload)

# 新代码（修复）
return templates.TemplateResponse(request, name, payload)
# 第一个参数需要传入 request 对象
```

### 问题 6: admin 密码哈希为空
**现象**: 登录时密码验证失败
**原因**: 数据库中 admin 用户的 password_hash 为 NULL
**解决**: 用脚本直接更新数据库
```python
# 生成正确哈希并更新
new_hash = pwd_context.hash("ChangeMe123!")
# UPDATE users SET password_hash = 'xxx' WHERE username = 'admin'
```

### 问题 7: 数据库乱码
**现象**: 导入后数据显示为 ������
**原因**: SQLite 数据库编码问题或字段映射错误
**解决**: 清空数据库重新导入

### 问题 8: 导入后数据不显示
**现象**: 导入成功但页面空白
**原因**: 
- 导入数据的 settlement_period = "2026-01"（旧数据）
- 页面默认显示当前月 "2026-05"
**解决**: 
- 修改导入逻辑：空日期时用当前月
- 或修改页面查询：显示所有月份数据

### 问题 9: 导入重复数据报错
**现象**: `IntegrityError: UNIQUE constraint failed`
**原因**: 数据库有 (owner_id, name, settlement_period) 唯一约束
**解决**: 导入前检查组合是否已存在
```python
existing = db.scalar(select(Person).where(...))
if existing:
    continue  # 跳过
```

### 问题 10: 后台进程启动失败
**现象**: 前台启动正常，后台启动后端口未监听
**原因**: paramiko session 关闭时带走子进程
**解决**: 使用 `nohup` + `&` 或写入启动脚本
```bash
nohup python -m uvicorn app.main:app --host 0.0.0.0 --port 8888 --workers 2 > app.log 2>&1 &
```

## 五、经验教训

1. **版本兼容性很重要**: Jinja2 3.0 + Starlette 1.0 的 API 变化导致模板渲染失败
2. **PYTHONPATH 必须设置**: 否则 `from app.xxx` 导入会失败
3. **后台启动要用 nohup**: 否则 SSH 断开进程就死了
4. **代理不稳定时直接连**: SOCKS5 代理经常超时
5. **数据库初始化要检查**: admin 密码哈希不能为空
6. **SFTP 比 Git 更可靠**: 代理不稳定时直接传文件
